# Infrastructure / Database Backup Implementation

## Overview

A PostgreSQL database backup and recovery process was implemented for the Fitness Assistant application. The goal of this process was to create a reliable method for backing up the production database hosted on Render and restoring it to a PostgreSQL database running on a Raspberry Pi.

This workflow demonstrates the ability to:

* Create a database backup from a production PostgreSQL environment.
* Securely transfer backup files between systems.
* Restore a database backup into another PostgreSQL instance.
* Verify that database tables and data were successfully recovered.

The backup workflow is:

```
Render PostgreSQL Database
          |
          | pg_dump
          v
Database Backup File (.sql)
          |
          | scp transfer
          v
Raspberry Pi Backup Server
          |
          | PostgreSQL restore
          v
Local Backup Database
```

---

# Backup Creation Using pg_dump

## Purpose

`pg_dump` is a PostgreSQL utility used to export a database into a SQL backup file. The generated file contains the commands needed to recreate database tables, relationships, indexes, and stored data.

Example command:

```bash
pg_dump --no-owner "RENDER_DATABASE_CONNECTION_STRING" > fitness_backup.sql
```

## Explanation

* `pg_dump` connects to the production PostgreSQL database.
* The database connection string identifies the Render database.
* `--no-owner` makes the backup portable by preventing ownership conflicts when restoring to another PostgreSQL server.
* `>` redirects the output into a file named `fitness_backup.sql`.

The resulting file is a snapshot of the database that can be restored later.

Example output:

```
fitness_backup.sql
Size: 11 KB
```

# Secure File Transfer Using SCP

## Purpose

The generated backup file was transferred from the development computer to the Raspberry Pi using SCP (Secure Copy Protocol).

Example command:

```bash
scp fitness_backup.sql holly@192.168.0.137:~/db_backups/
```

## Explanation

* `scp` securely copies files between computers.
* `fitness_backup.sql` is the local backup file.
* `holly@192.168.0.137` identifies the Raspberry Pi.
* `~/db_backups/` is the destination directory on the Pi.

After transfer, the backup file was stored on the Raspberry Pi:

```
/home/holly/db_backups/fitness_backup.sql
```

Verification:

```bash
ls -lh ~/db_backups
```

Result:

```
-rw-r--r-- 1 holly holly 11K fitness_backup.sql
```

# Database Restore on Raspberry Pi

## Purpose

The SQL backup file was restored into a PostgreSQL database running on the Raspberry Pi.

Example command:

```bash
psql -U fitness -d fitness_assistant_backup < ~/db_backups/fitness_backup.sql
```

## Explanation

* `psql` is the PostgreSQL command-line client.
* `-U fitness` connects using the PostgreSQL user created for the backup database.
* `-d fitness_assistant_backup` specifies the destination database.
* `< fitness_backup.sql` sends the SQL commands from the backup file into PostgreSQL.

The restore process recreated:

* Database tables
* Relationships
* Indexes
* Stored records

Here is the terminal readout showing my code creating my backup database and my cron script that schedules when the create backup database code runs which is 2am every morning. Then I have a command that shows the output displaying all the backups I have saved in my backups folder. 

holly@holly-pi:~ $ cat ~/scripts/backup_fitness_db.sh
#!/bin/bash

# Create timestamp
DATE=$(date +"%Y-%m-%d_%H-%M")

# Backup location
BACKUP_DIR="/home/holly/db_backups"

# Filename
BACKUP_FILE="$BACKUP_DIR/fitness_backup_$DATE.sql"

# Database connection information
HOST="dpg-d9cli261a83c739b3tkg-a.oregon-postgres.render.com"
USER="the_fitness_assistant_user"
DATABASE="the_fitness_assistant"

# Create backup
/usr/lib/postgresql/18/bin/pg_dump \
--no-owner \
-h "$HOST" \
-U "$USER" \
-d "$DATABASE" \
-f "$BACKUP_FILE"

# Check success
if [ $? -eq 0 ]; then
    echo "Backup completed: $BACKUP_FILE"

    # Delete backups older than 30 days
    find "$BACKUP_DIR" \
    -name "fitness_backup_*.sql" \
    -mtime +30 \
    -delete

    echo "Old backups cleaned up."

else
    echo "Backup failed!"
    rm -f "$BACKUP_FILE"
fi
